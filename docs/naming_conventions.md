# Naming Conventions

Standards and patterns used throughout the NFIP Insurance Data Warehouse project.

---

## Database Schemas

| Schema | Layer | Purpose |
|--------|-------|---------|
| `bronze` | Bronze | Raw staging — all VARCHAR, no transformation |
| `silver` | Silver | Cleaned, typed, deduplicated, enriched |
| `gold` | Gold | Star schema — dimensions and facts for analytics |

Schema names are **lowercase**, single-word.

---

## Table Naming

| Layer | Pattern | Example |
|-------|---------|---------|
| Bronze | `{schema}.nfip_{entity}_raw` | `bronze.nfip_claims_raw` |
| Silver | `{schema}.nfip_{entity}_cleaned` | `silver.nfip_claims_cleaned` |
| Silver (ref) | `{schema}.ref_{entity}` | `silver.ref_state_info` |
| Gold (dim) | `{schema}.dim_{entity}` | `gold.dim_location` |
| Gold (fact) | `{schema}.fact_{entity}` | `gold.fact_claims` |
| Gold (view) | `{schema}.vw_{metric}` | `gold.vw_loss_ratio_by_state` |

All table names use **snake_case**.

---

## Column Naming

| Layer | Convention | Rationale |
|-------|------------|-----------|
| Bronze | **camelCase** | Matches CSV headers from the OpenFEMA API exactly for BULK INSERT compatibility |
| Silver | **camelCase** (source) + **snake_case** (derived) | Source columns retain API names; derived columns like `zone_category` use snake_case |
| Gold | **snake_case** | All columns use snake_case for consistency in the analytics layer |

---

## Key Columns

| Type | Pattern | Example |
|------|---------|---------|
| Surrogate key (dim) | `{entity}_key` | `location_key`, `flood_zone_key` |
| Surrogate key (fact) | `{entity}_sk` | `claim_sk`, `policy_sk` |
| Date key (FK) | `{role}_date_key` or `date_key` | `effective_date_key`, `termination_date_key` |
| Unknown row | Key value = **-1** | `location_key = -1` → state 'ZZ', name 'Unknown' |

Dimension keys use `_key` suffix. Fact surrogate keys use `_sk` suffix. The Unknown row at key `-1` is present in every dimension to handle NULL or unresolvable lookups from fact tables.

---

## SQL Script Files

| Layer | Pattern | Example |
|-------|---------|---------|
| Bronze | `scripts/bronze/{NN}_{action}.sql` | `01_create_database.sql`, `03_load_claims.sql` |
| Silver | `scripts/silver/{NN}_{action}.sql` | `02_clean_claims.sql`, `05_validate_silver.sql` |
| Gold | `scripts/gold/{NN}_{action}.sql` | `01_create_gold_tables.sql`, `09_create_analytics_views.sql` |
| Tests | `tests/test_{domain}.sql` | `test_row_counts.sql`, `test_business_rules.sql` |

Scripts are **numbered** with a two-digit prefix to enforce execution order. All filenames use **snake_case**.

---

## Python Files

| Location | Pattern | Example |
|----------|---------|---------|
| Ingestion | `scripts/ingest/{action}.py` | `fetch_nfip_claims.py`, `profile_data.py` |
| Fallback | `scripts/bronze/{action}.py` | `load_via_python.py` |

Python filenames use **snake_case**. Constants in Python scripts use **UPPER_SNAKE_CASE**.

---

## Data Files (CSV)

| Dataset | Pattern | Example |
|---------|---------|---------|
| Claims | `datasets/claims/{STATE}_claims.csv` | `FL_claims.csv`, `NY_claims.csv` |
| Policies | `datasets/policies/{STATE}_policies.csv` | `FL_policies.csv`, `TX_policies.csv` |

State abbreviation is **uppercase**, two letters. CSV files are gitignored and generated by the ingestion scripts.

---

## Shell Scripts

| Location | Pattern | Example |
|----------|---------|---------|
| Helpers | `scripts/{action}.sh` | `docker_start.sh`, `run_sql.sh` |

Shell scripts use **snake_case** and are marked executable (`chmod +x`).
